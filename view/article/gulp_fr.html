Gulp est ce qu'on appelle un "Javascript task-runner" ou en français "Automatiseur de tâches", ce qui veut dire que ce toolkit est capable d'automatiser des tâches répétitives et d'accélérer le développement d'un projet à l'aide de plugins : principalement du côté front-end.

Ses tâches sont aussi utiles que différentes les unes des autres :

  • Créer un serveur local.
  • Rechargement automatique du navigateur.
  • Optimisation et compression d'images.
  • Minifier et optimiser du CSS ou du JavaScript.
  • Création ou suppression de dossiers ou de fichiers.
  • Etc ...

Il existe actuellement près de 4000 plugins qui, une fois choisis et intégrés à votre projet, vous feront gagner énorménent de temps de production.

Je vous propose de vous montrer comment implémenter Gulp à vos projets suivi d'un exemple.

C'est parti :

Pour commencer, vous aurez besoin d'installer Node.js. C'est un environnement Javascript qui est équipé de plugins nommés "npm". Nous aurons besoin de ce dernier pour mettre en place notre environnement Gulp.

Une fois Node.js installé, ouvrez un terminal de commande (cmd ou powershell) n'importe où :

npm install --global gulp-cli

Cette commande aura pour effet d'installer Gulp de manière globale sur votre ordinateur et ce une fois pour toutes.

Nous allons maintenant initialiser gulp dans vos projets. Ouvrez un terminal de commande à la racine de votre projet :

npm init

On vous demandera d'entrer plusieurs informations sur votre projet : nom, version, description, etc.. Ne vous en faites pas, vous pourrez modifier ces informations par la suite (dans le fichier package.json).

Il en suit, la partie la plus importante et la plus amusante : le choix des plugins et leur installation. Pour ce faire direction https://gulpjs.com/plugins

Une fois votre choix fait, il faudra installer l'ensemble des plugins via votre terminal :

npm install --save-dev

Ils devront être entrés un à un.

Si vous souhaitez ne pas vous prendre la tête pour le moment, je vous propose d'installer les modules principaux de Gulp :

npm install gulp --save-dev

Une fois fait, un dossier `node_modules` devrait apparaître dans votre projet. Voilà, il ne reste plus qu'a créer vos automatisations.

Pour cela, vous allez devoir créer un fichier à la racine de votre projet: gulpfile.js. Il sert à gérer les tâches, leurs options, leurs sources et destinations. Dans celui-ci, commencez par déclarer toutes les variables dont vous aurez besoin.

// Requis
var gulp = require('gulp');

// Include plugins
var plugins = require('gulp-load-plugins')();
        // tous les plugins de package.json

// Variables de chemins
var source = './src';           // dossier de travail
var destination = './dist';     // dossier à livrer.

À présent, dans ce même fichier, nous allons créer notre première tâche : minifier le css

gulp.task('minifier', function() {       // Tâche "minifier" 
    return gulp.src(source + *.css)      // Copie du fichier à modifier 
    .pipe(plugins.csso())                // Application du plugin sur le fichier 
                                         // Ici plus de plugins Gulp à exécuter 
    .pipe(gulp.dest(destination));       // Destination du fichier modifié
});

Pour exécuter cette tâche, il vous faudra rouvrir votre terminal dans votre projet :

gulp minifier

Et, ainsi, quand vous commencerez à avoir plusieurs tâches, vous pourrez les regrouper et ainsi, créer plusieurs zones de développement.

Par exemple :

// Tâche "build"
gulp.task('build', ['tache']);

// Tâche "prod" = Build + minifier
gulp.task('prod', ['build', 'minifier']);

Voilà, j'espère que cette introduction a Gulp vous a plu et que votre curiosité est satisfaite. Quoi qu'il en soit, l'usage de cet outil améliorera et optimisera votre façon de travailler.

Bon code.
